What abliteration moved

We fit two Jacobian lenses on the same corpus — one on stock Qwen3.5-27B, one on its abliterated (heretic) twin — then subtracted them. The diff recovers the edit: it localises where abliteration changed the transport (the residual-writing matrices, §05) and reads out its direction — a safety/refusal signature in the mid-to-late band. The edit is a static directional weight change, so its magnitude is prompt-independent (same size on any input) — but because a fixed direction is read through prompt-dependent residuals, its token-space content is prompt-conditional: safety vocab on safety prompts, formatting noise on a benign one (§04). A modest, honest result: the lens sees what abliteration did and where.

01

Two lenses, one shared substrate — vary only the map

A lens is just a linear map J_l per band layer. To turn it into token read-outs you apply it to real residual activations — and those come from running a model. That model is the substrate. The trick of the diff: hold the substrate fixed and identical for both lenses, so their read-outs can only differ by the transport geometry each lens learned. Same activations in; different map.

why the corpus is shared too Both lenses were fit on the same token sequences (item 10 dropped in both). So the only thing that differs between A and B is the model that produced the transport — not the data it saw, not the substrate it's read on. Cleanest possible one-variable comparison.

04

Why more safety, not less — abliteration edits the transport, not the read-out

The naive guess: abliteration removes refusal, so B − A should show less safety. It shows the opposite — and the weight footprint (§05) says exactly where and how. Heretic performs directional ablation: it orthogonalises the residual-writing matrices — every layer's attn.o_proj and mlp.down_proj — against the refusal direction r = mean(harmful) − mean(harmless). Those matrices are the tail blocks, which sit inside J. The read-out (norm, head, softcap) it leaves untouched: model.norm is bit-identical between the two models, lm_head and embed at the quant-noise floor.

└─ inside J — abliteration edits this (transport) ─┘ └─ outside J — bit-identical (read-out) ─┘

So the edit is inside the map, and the read-out is held fixed — which is precisely why the lens is the right instrument: the diff B − A is a pure transport difference, uncontaminated by any read-out change. What it reads out is the direction of the weight edit itself: abliteration removed the refusal-behaviour axis (harmful-vs-harmless means) from o_proj/down_proj, and that removal has a fixed signature in vocabulary space — safety/refusal tokens. The edit's magnitude is the same on any input (it's a static weight change); what the lens reads out in token space depends on the prompt, because the fixed direction is projected through prompt-dependent residuals.

clean-pair test (2026-07-14): magnitude flat, content prompt-conditional — floor HOLDS The per-prompt run on the clean matched pair (`pair-base` vs `pair-heretic-t144`) splits the question in two. Magnitude is prompt-independent: mean l2 over L32–42 is essentially flat — benign recipe 452, safety:lock 457, safety:chemicals 460 (a ~1% margin). But content is prompt-conditional: at L34 the safety prompts surface pure safety vocab (安全风险 Safety 安全性 不承担任何) while the benign recipe surfaces pure formatting (** Nothing) — and a benign-but-security-adjacent prompt (crypto history) leaks a little safety vocab at rank 6–8. So there is a content floor: a truly benign prompt does not surface safety words.

This reverses the older amber note (below), which ran on the uncontrolled pair (mlx-community stock vs coder3101, differing in converter and recipe) and read benign-lights-up as "no floor." That l2-only reading conflated magnitude with content; on a controlled pair the two separate cleanly.
superseded — original-pair note (2026-07-13) On the original pair a benign "weeknight recipe" prompt lit up L32–42 as strongly as the safety prompts (mean l2 596 vs ~524–571) and surfaced Nothing/cannot/unsafe, read at the time as "no benign floor, uniform across content." Kept for the record; the clean-pair test above is the corrected version.
what the tokens still tell us The surfaced tokens — unethical, illegal, harmful, violat… — are almost verbatim Heretic's own refusal-keyword list, the words it optimises the model to stop emitting. So the diff correctly characterises what abliteration changed (a refusal-direction edit) and where (§05, same layers as the weight edit) — it just doesn't imply a content-conditional disposition.
05

The weight footprint — an independent second measurement, and it agrees

Everything above reads the transport (the Jacobian). Here's a completely independent view: dequantise both 8-bit models and take the relative weight delta ‖WB − WA‖ / ‖WA‖ per layer. This measures where abliteration changed the weights — no lens involved. If it peaks where the transport diff peaks, two independent methods agree on where abliteration lives.

weight delta per layer (all 64) lens band 16–47 transport safety cluster 32–42
the cross-validation The weight edit peaks at L33 (0.039) and L36 (0.034) and plateaus L25–50 — and the transport-diff safety cluster (§02) peaks at the same L33–37, spanning L32–42. Two measurements that share no machinery — weight-space edit magnitude vs transport-space read-out shift — land on the same layers. That co-localisation is what turns "suggestive" into "mechanistic".

And which matrices? The edit is ~6× concentrated in the three residual-writing projections, exactly as directional ablation prescribes — the input matrices (q/k/v, gate, up) sit at the quant floor:

residual-writing (o_proj / down_proj) input matrices (floor)
confirmed against the Heretic source This isn't inferred — Heretic's code orthogonalises exactly attn.o_proj + mlp.down_proj (skipping embed/head), on a per-layer tent schedule (max_weight at max_weight_position, ramping to zero past min_weight_distance). The footprint above is that tent, imaged from the weights: flat → ramp → plateau/peak L33–36 → taper. And the vision tower is bit-identical (max |B−A| = 0) — abliteration never touched it.
06

It isn't a substrate artifact — both directions agree

Which model's residuals you feed as the substrate is a free choice, and it could bias the result — so I ran it both ways: once reading on the stock model's activations, once on the abliterated model's. If the finding flipped, it'd be an artifact of the input. It didn't — the safety cluster comes out layer-for-layer identical. The l2 per layer, L32–L42, stock (teal) vs heretic (indigo) substrate:

stock substrate heretic substrate
what agreement buys you Because both substrates give the same answer, the finding is a property of the lens pair — the difference in transport geometry — not of whose activations happened to be the input. That's the check the diff script explicitly asks for, and it passed.
07

What it does — and does not — establish

One limit that flipped once we controlled the pair, and two still open. The finding this leaves standing is solid: the diff recovers abliteration's edit — its location (§05) and its vocabulary-space direction. The edit's magnitude is prompt-independent; its token-space content is prompt-conditional (§04).

Content floor — HOLDS on the clean pair

On the uncontrolled pair a per-prompt run read "no benign floor" (benign recipe l2 596 ≈ safety, surfacing refusal vocab). The clean matched pair reverses it: l2 magnitude is flat across prompts (benign 452 ≈ safety 457–460), but the benign recipe's top tokens are pure formatting while safety prompts surface safety vocab. Magnitude prompt-independent, content prompt-conditional — the floor is real (§04). The old falsification confounded converter + recipe.

Quant match unconfirmed

Both are 8-bit but possibly different converter versions. The illegible early-band *-junk / CJK sludge is plausibly quant noise. The mid-late safety cluster is too coherent and too consistent across both substrates to be quant noise — so the headline survives — but discount early-band magnitude accordingly.

By eye, not by score

This finding rests entirely on the read-out tokens. The per-layer legibility metric misleads (established on band-n14-fixed: it ranked degenerate deep layers 0.91–0.93 over the meaningful L40 at 0.85). A genuinely disposition-aware metric is still open.

08

Reproduce it — CLI now, heylook later

Everything above came from two commands. Both are Metal-gated (stop the server first) and must run unsandboxed. When the heylook jspace page lands, this is the exact pipeline it wraps — the lens dirs and read-out semantics are identical, so a CLI run and a UI run line up one-to-one.

1 · fit the stock counterpart on the same corpus JLENS_MODEL=…/Qwen3.5-27B-8bit-mlx JLENS_OUT=out/band-n14-stock \
  JLENS_LAYERS=16,…,47 JLENS_N=14 ./scripts/fit_band_supervisor.sh

Copy band-n14-fixed/ckpt/corpus.json into the stock ckpt/ first — the fit reuses those token ids instead of regenerating, so the corpus is identical. It resumes, skips generation, and finishes clean (identity_ok: true).

2 · diff the two lenses on a shared substrate JLENS_MODEL=…/Qwen3.5-27B-8bit-mlx \
  JLENS_LENS_A=out/band-n14-stock JLENS_LENS_B=out/band-n14-fixed \
  JLENS_TOPK=10 uv run python scripts/diff_lenses.py

Swap JLENS_MODEL to the -heretic- path to run the other substrate direction. A = stock, B = abliterated; top_up = what B surfaces more, top_down = what B suppresses.

what to look for when you run it

Ignore the tallest early-band bars — they're *-junk. Scroll to L32–L42: if top_up there reads Safety / unsafe / 安全风险 / Cannot / violations, you're seeing the same transport-level refusal signal we did. The benign prompt's rows should be near-flat; the safety-adjacent prompts carry the delta. That alignment — CLI read-out ↔ what the model won't say — is the whole point of the lens.
